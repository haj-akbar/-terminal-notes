# Terminal Learning Journey

این فایل را درست کردم تا دستورهای پایه‌ی لینوکس را یک‌جا داشته باشم و بعداً راحت‌تر مرورشان کنم.

بیشتر چیزهایی که اینجا نوشتم، دستورهایی هستند که موقع کار با ترمینال زیاد استفاده می‌شوند.

---

## 0. The Typical Command Format

اکثر دستورهای ترمینال تقریباً یک شکل کلی دارند:

command [option] [target]

مثلاً:


ls -la /home

در این دستور:


command: ls
option: -la
target: /home

یعنی دستور اصلی `ls` است، گزینه‌ی `-la` رفتار دستور را تغییر می‌دهد، و `/home` مسیری است که دستور روی آن اجرا می‌شود.

---

## 1. pwd

دستور `pwd` بهم نشان می‌دهد الان دقیقاً داخل کدام مسیر هستم.

بعداً فهمیدم `pwd` مخفف `Print Working Directory` است.


pwd

### مسیر مطلق و نسبی

مسیر مطلق از ریشه‌ی سیستم شروع می‌شود:


/home/ali/projects

ولی مسیر نسبی نسبت به جایی حساب می‌شود که الان داخلش هستیم:


projects/file.txt

این تفاوت برایم مهم بود، چون خیلی وقت‌ها دستورها به مسیر فایل یا پوشه وابسته هستند.

---

## 2. cd

دستور `cd` برای رفتن از یک پوشه به پوشه‌ی دیگر استفاده می‌شود.

`cd` مخفف `Change Directory` است.


cd folder-name

چند مسیر پرکاربرد:

text
.      پوشه‌ی فعلی
..     یک پوشه عقب‌تر
~      پوشه‌ی Home کاربر
/      ریشه‌ی سیستم

مثال:


cd ..

cd ~

cd /home


---

## 3. ls

دستور `ls` فایل‌ها و پوشه‌های داخل مسیر فعلی را نشان می‌دهد.

`ls` مخفف `List` است.


ls

چند حالت پرکاربرد:


ls -l

نمایش جزئیات بیشتر.


ls -a

نمایش فایل‌های مخفی.


ls -la

نمایش همه‌ی فایل‌ها همراه با جزئیات.


ls -lh

نمایش حجم فایل‌ها به شکل خواناتر(humanity readable).

من معمولاً از این دستور زیاد استفاده می‌کنم:


ls -alh

چون هم فایل‌های مخفی را نشان می‌دهد، هم اطلاعات کامل‌تری می‌دهد.

---

## 4. mkdir

دستور `mkdir` برای ساخت پوشه‌ی جدید است.

`mkdir` مخفف `Make Directory` است.


mkdir folder1

برای ساخت چند پوشه با هم:

mkdir folder1 folder2 folder3

اگر بخواهم پوشه‌های تو در تو بسازم، از `-p` استفاده می‌کنم:

mkdir -p me/university/homework

این گزینه باعث می‌شود اگر پوشه‌های وسط مسیر وجود نداشتند، خود دستور آن‌ها را هم بسازد.

---

## 5. touch

دستور `touch` معمولاً برای ساخت فایل خالی استفاده می‌شود.


touch name.txt

برای ساخت چند فایل با هم:


touch name1.md name2.txt name3.sh

ساخت فایل داخل یک مسیر مشخص:


touch me/university/homework/project1.txt

این دستور برای تمرین و ساخت سریع فایل خیلی به درد می‌خورد.

---

## 6. echo

دستور `echo` متن را در ترمینال چاپ می‌کند.


echo "Hello"

می‌شود با آن داخل فایل هم متن نوشت:


echo "متن مورد نظر" > name.txt

نکته‌ی مهم این است که علامت `>` محتوای قبلی فایل را پاک می‌کند و متن جدید را جایگزین می‌کند.

اگر بخواهم متن جدید را به انتهای فایل اضافه کنم، از `>>` استفاده می‌کنم:


echo "متن جدید" >> name.txt

برای نمایش مقدار یک متغیر هم از `$` قبل از اسم متغیر استفاده می‌شود:


echo $HOME

---

## 7. cat

دستور `cat` محتوای فایل را سریع در ترمینال نشان می‌دهد.

cat file.txt

برای فایل‌های کوچک خوب است، ولی اگر فایل خیلی بزرگ باشد، خروجی زیادی در ترمینال چاپ می‌شود.

برای همین در فایل‌های بزرگ‌تر بهتر است از `head`، `tail` یا `less` استفاده شود.

### head و tail

`head` به صورت پیش‌فرض ۱۰ خط اول فایل را نشان می‌دهد:


head file.txt

برای تعداد مشخصی از خط‌ها:


head -n 5 file.txt

`tail` هم به صورت پیش‌فرض ۱۰ خط آخر فایل را نشان می‌دهد:


tail file.txt

برای تعداد مشخصی از خط‌های آخر:


tail -n 5 file.txt

---

## 8. less

دستور `less` برای خواندن فایل‌های بزرگ بهتر از `cat` است، چون کل فایل را یک‌دفعه در ترمینال نمی‌ریزد.


less file.txt

وقتی داخل `less` هستم، این کلیدها کاربرد دارند:


Space    رفتن به صفحه‌ی بعد

b        برگشتن به صفحه‌ی قبل

/word    جست‌وجوی یک کلمه

n        رفتن به نتیجه‌ی بعدی جست‌وجو

q        خروج

مثلاً برای خروج فقط کافی است کلید `q` را بزنم.

---

## 9. cp

دستور `cp` برای کپی کردن فایل یا پوشه استفاده می‌شود.

`cp` مخفف `Copy` است.


cp name.txt docs/new-name.txt

در این مثال، فایل `name.txt` داخل پوشه‌ی `docs` کپی می‌شود و اسم نسخه‌ی کپی‌شده `new-name.txt` می‌شود.

اگر نخواهم اسم فایل را عوض کنم، فقط مسیر پوشه را می‌نویسم.

برای کپی کردن پوشه و محتویاتش باید از `-r` استفاده کنم:


cp -r project project-backup

---

## 10. mv

دستور `mv` هم برای جابه‌جا کردن استفاده می‌شود، هم برای تغییر نام.

`mv` مخفف `Move` است.

تغییر نام فایل:

mv name1.txt name2.txt

جابه‌جا کردن فایل یا پوشه:


mv task docs/

انتقال به مسیر فعلی:


mv task .


---

## 11. rm

دستور `rm` برای حذف فایل یا پوشه استفاده می‌شود.

`rm` مخفف `Remove` است.


rm text.txt

حذف چند فایل:


rm text.txt docs/q.md

چند گزینه‌ی مهم:


-r     حذف پوشه و تمام محتویات آن

-i     گرفتن تأیید قبل از حذف

-rf    حذف اجباری و بدون پرسش

برای حذف پوشه‌ی خالی می‌شود از `rmdir` استفاده کرد:


rmdir empty-folder

نکته‌ی خیلی مهم: دستور `rm -rf` خطرناک است، چون بدون پرسیدن می‌تواند فایل‌ها و پوشه‌ها را حذف کند.

---

## 12. grep

دستور `grep` برای پیدا کردن یک کلمه یا عبارت داخل فایل‌ها استفاده می‌شود.


grep "word" name.txt

گزینه‌های پرکاربرد:


-i      بدون حساسیت به بزرگی و کوچکی حروف

-n      نمایش شماره خط

-r      جست‌وجوی بازگشتی داخل پوشه‌ها

*.txt   جست‌وجو در چند فایل متنی

مثال:


grep -n "error" log.txt

جست‌وجو داخل همه‌ی فایل‌های متنی:


grep "hello" *.txt

جست‌وجو داخل پوشه‌ها و زیرپوشه‌ها:

grep -r "password" .

این دستور برای پیدا کردن سریع متن داخل فایل‌ها خیلی کاربردی است.

---

## 13. find

دستور `find` برای پیدا کردن فایل‌ها و پوشه‌ها استفاده می‌شود.


find . -name "test.txt"

در این مثال، جست‌وجو از پوشه‌ی فعلی شروع می‌شود.

جست‌وجو بر اساس نام:


find . -name "todo.txt"

جست‌وجوی فقط فایل‌ها:


find . -type f -name "*.txt"

یعنی از پوشه‌ی فعلی شروع کن، فقط فایل‌ها را پیدا کن، و فقط آن‌هایی را نشان بده که با `.txt` تمام می‌شوند.

مثال دیگر:


find me/university -name "todo.txt"

یعنی داخل مسیر `me/university` دنبال چیزی به نام `todo.txt` بگرد.

---

## 14. chmod

دستور `chmod` برای تغییر مجوزهای فایل‌ها و پوشه‌ها استفاده می‌شود.

`chmod` مخفف `Change Mode` است.

برای دیدن مجوزها می‌شود از این دستور استفاده کرد:


ls -l

ممکن است خروجی شبیه این باشد:


-rw-r--r-- 1 ali ali 120 Jun 10 notes.txt

قسمتی که برای مجوزها مهم است:


-rw-r--r--

### نوع مجوزها


r    خواندن یا Read

w    نوشتن یا Write

x    اجرا کردن یا Execute

### گروه‌ها


owner     مالک فایل

group     گروه فایل

others    سایر کاربران

مثلاً:


-rw-r--r--

یعنی:


rw-    مالک می‌تواند بخواند و بنویسد

r--    گروه فقط می‌تواند بخواند

r--    سایر کاربران فقط می‌توانند بخوانند

### روش حروفی

فرم کلی:

chmod [u/g/o/a][+/-][rwx] file-name

معنی حروف:


u    مالک یا User

g    گروه یا Group

o    سایرین یا Others

a    همه یا All

مثال:


chmod u+x hello.sh

یعنی به مالک فایل اجازه‌ی اجرا بده.

مثال دیگر:


chmod g-w notes.txt

یعنی اجازه‌ی نوشتن را از گروه بگیر.

### روش عددی

در روش عددی از یک عدد سه‌رقمی استفاده می‌شود.


4 = خواندن
2 = نوشتن
1 = اجرا
0 = بدون دسترسی

عددها با هم جمع می‌شوند:


7 = 4 + 2 + 1 = rwx

6 = 4 + 2     = rw-

5 = 4 + 1     = r-x

4 = 4         = r--

0 = 0         = ---

مثال:


chmod 755 hello.sh

یعنی مالک دسترسی کامل دارد، اما گروه و سایر کاربران فقط اجازه‌ی خواندن و اجرا دارند.


chmod 644 notes.txt

یعنی مالک اجازه‌ی خواندن و نوشتن دارد، اما گروه و سایر کاربران فقط اجازه‌ی خواندن دارند.


chmod 700 private.sh

یعنی فقط مالک دسترسی کامل دارد.

### کاراکتر اول در خروجی ls -l

در خروجی `ls -l`، کاراکتر اول نوع فایل را نشان می‌دهد:


-    فایل معمولی
d    پوشه یا Directory

l    لینک نمادین یا Symbolic Link

c    Character Device

b    Block Device

p    Named Pipe

s    Socket

مثال:


drwxr-xr-x

یعنی این مورد یک پوشه است.

مثال دیگر:


-rw-r--r--

یعنی این مورد یک فایل معمولی است.

## 15. man

دستور `man` راهنمای کامل دستورها را نشان می‌دهد.

`man` مخفف `Manual` است.


man command

مثال:


man ls

یعنی راهنمای دستور `ls` را باز کن.

وقتی داخل صفحه‌ی `man` هستم، چند کلید مهم وجود دارد:


Space    رفتن به صفحه‌ی بعد

b        برگشتن به صفحه‌ی قبل

/word    جست‌وجوی یک کلمه

n        رفتن به نتیجه‌ی بعدی

q        خروج

مهم‌ترینش برای من `q` است، چون با آن از صفحه‌ی راهنما خارج می‌شوم.
