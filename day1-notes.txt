Summary of Necessary Linux Commands

0. The Typical Command Format

    command [options] [target]

    A typical Linux command usually has three parts:
    - command: the main command
    - options: extra settings for the command
    - target: the file or folder that the command works on


1. pwd

    pwd means Print Working Directory.

    It shows the current directory that you are in.


2. cd

    cd means Change Directory.

    It is used to move between directories.

    Important signs:

    ..    Go to the parent directory
    .     Current directory
    ~     Home directory
    /     Root directory


3. ls

    ls is used to list files and folders.

    Important options:

    -l    Shows details
    -a    Shows hidden files
    -la   Shows all files with details

    Notice:

    When you use ls -l, you may see something like this:

    drwxr-xr-x  5 ali ali 4096 Jun 10 Desktop

    If the first character is d, it means the item is a directory.
    If the first character is -, it means the item is probably a file.


4. mkdir

    mkdir means Make Directory.

    It is used to create new directories.

    Basic usage:

    mkdir folder1 folder2

    Notice:

    To create nested directories, use the -p option.

    Example:

    mkdir -p folder1/folder2/folder3


5. touch

    touch is used to create a new empty file.

    Example:

    touch file.txt


6. echo

    echo is used to print text or write text into a file.

    Important signs:

    >     Replaces the old content of the file
    >>    Adds new content to the end of the file

    Examples:

    echo "Hello" > file.txt
    echo "New line" >> file.txt


7. cat

    cat is used to open and display the content of a small file.

    Example:

    cat file.txt


8. less

    less is used to open and read large files.

    Important keys:

    Space    Next page
    b        Previous page
    /word    Search for a word
    q        Quit


9. cp

    cp means Copy.

    It is used to copy files or folders.

    Basic usage:

    cp source_file destination_folder/new_name.txt

    Notice:

    The new name is optional.

    Important option:

    -r    Copies a folder with all of its contents

    Example:

    cp -r folder1 folder2


10. mv

    mv means Move.

    It is used to move or rename files and folders.

    To rename a file:

    mv old_name.txt new_name.txt

    To move a file to another folder:

    mv file.txt folder/


11. rm

    rm means Remove.

    It is used to remove files or folders.

    Examples:

    rm file.txt
    rm file1.txt file2.txt

    Important options:

    -r     Removes a folder and all of its contents
    -i     Asks before deleting
    -rf    Removes a folder and all of its contents without asking

    Notice:

    Be careful with rm -rf because it deletes files without confirmation.


12. grep

    grep is used to search for a word or text inside files.

    Basic usage:

    grep "word" name.txt

    Important options:

    -i       Ignores capital and small letters
    -r       Searches recursively inside folders
    -n       Shows the line number
    *.txt    Searches inside all txt files in the current folder

    Examples:

    grep -i "hello" name.txt
    grep -n "hello" name.txt
    grep "hello" *.txt
    grep -r "hello" .


13. find

    find is used to find files and folders.

    Basic usage:

    find .

    This command finds everything in the current directory.

    Important options:

    -name    Searches by name
    -type    Searches by type

    Examples:

    find . -name "file.txt"
    find . -name "*.txt"
    find . -type f -name "*.txt"
    find docs/ -type f -name "*.txt"

    Explanation:

    -type f means files only.
    -type d means directories only.
    *.txt means all files that end with .txt.


14. chmod

    chmod means Change Mode.

    It is used to change file and folder permissions.

    When you use ls -l, you may see something like this:

    -rw-r--r--

    The important part is:

    rw-r--r--

    Permissions are divided into three groups:

    1. Owner
    2. Group
    3. Others

    There are three types of permissions:

    r    Read
    w    Write
    x    Execute

    Example:

    -rw-r--r--

    This means:

    Owner: rw-    The owner can read and write
    Group: r--    The group can only read
    Others: r--   Others can only read

    Symbolic method:

    chmod u+x file.sh

    This means: add execute permission to the owner.

    Important letters:

    u    User / Owner
    g    Group
    o    Others
    a    All

    Important signs:

    +    Add permission
    -    Remove permission

    Numeric method:

    Each permission has a number:

    4    Read
    2    Write
    1    Execute
    0    No permission

    Example:

    chmod 750 file.sh

    Explanation:

    7 = 4 + 2 + 1 = read, write, execute
    5 = 4 + 1 = read, execute
    0 = no permission

    So chmod 750 means:

    Owner has all permissions.
    Group has read and execute permissions.
    Others have no permissions.

    Other examples:

    chmod 755 file.sh
    chmod 644 file.txt
    chmod 700 private.sh


15. man

    man means Manual.

    It is used to read the manual page of a command.

    Example:

    man ls

    To quit the manual page, press q.
